# REG-FOCUS : Regulatory variants – Functional Outcome Classifier Using Sequencing–chromatin fusion

## Introduction
REG-FOCUS is a sequence-to-function model framework designed to predict regulatory effects of genetic variants across celltypes.  
![REG-FOCUS Model Schema](REG-FOCUS_model_schema.png)
Experimentally resolved regulatory effects were integrated with biology-guided sequence and chromatin representations through transfer learning to capture generalizable relationships between sequence perturbation, cellular context, and regulatory effects. The resulting framework predicts regulatory effects for previously unseen variants and cell types.

## Model 
### Pre-training
We used the DNABERT-2 model for sequence representation learning. The model is available from the official repository:https://github.com/MAGICS-LAB/DNABERT_2
The pre-trained weights can be automatically downloaded using the Hugging Face `transformers` library:
```
import torch
from transformers import AutoTokenizer, AutoModel
tokenizer = AutoTokenizer.from_pretrained("zhihan1996/DNABERT-2-117M", trust_remote_code=True)
model = AutoModel.from_pretrained("zhihan1996/DNABERT-2-117M", trust_remote_code=True)
```
We used the ChromBPNet bias model trained on scATAC-seq data from dermal fibroblasts to correct sequence-specific bias in chromatin accessibility prediction.
The bias model file is provided in this repository at:
```
./pre-trained_model/chrombpnet/bias_models/scATAC_dermal_fibroblast.h5
```

### DNA embedding and chromatin embedding
The models fine-tuned specifically on PBMC chromatin accessibility data is available at:
```
./fine-tuned_model/dnabert2/sc-PBMC
./fine-tuned_model/chrombpnet/sc-PBMC
```

### Transferable learning
The TabPFN model was trained on experimentally validated effect variants from CD14 monocytes and can be extended to other cell types.
The trained model is available in this repository at:
```
./classifier_model/tabpfn
```

## Quick Start
REG-FOCUS requires a Python 3.10 environment with all required dependencies installed. In the following examples, CD4 Naive T cells (CD4Naive) are used as a representative cell type.
**(1)Prepare the input variant file for prediction**
Create a TSV file containing the variants to be evaluated by REG-FOCUS.
Example:
> CD4Naive_single_SNP_for_pred.tsv
```
chr1	2146433	A	G	chr1:2146433:A:G
chr1	2153159	T	C	chr1:2153159:T:C
chr1	6007580	C	G	chr1:6007580:C:G
chr1	6490445	C	T	chr1:6490445:C:T
chr1	8510397	C	G	chr1:8510397:C:G
......
```

**(2)Generate allele-specific feature representations**
REG-FOCUS extracts allele-specific regulatory features using two fine-tuned models: DNABERT2 and ChromBPNet.
> Generate DNABERT2 features
```
python ./scripts/DNABERT2_testSNP_seq_prepare.py \
--ref ./data/hg38.fa \
CD4Naive_single_SNP_for_pred.tsv

python ./scripts/DNABERT2_testSNP_feature_pred.py \
  --model_path ./fine-tuned_model/dnabert2/sc-PBMC/dnabert2_finetune_CD4Naive/checkpoint-3600 \
  --input_dir ./SNP_for_pred \
  --output_file ./dnabert2_feature/SNP_for_pred_CD4Naive.csv \
  --max_length 120 \
  --batch_size 256
```
>Generate ChromBPNet features
```
bash ./scripts/ChromBPNet_testSNP_feature_pred.sh --cell CD4Naive --list CD4Naive_single_SNP_for_pred.tsv
```
The feature representations generated by the two models are then integrated at the variant level.

**(3) Predict regulatory effects**
The model assigns each variant to one of three regulatory outcome categories:
	•	up – increased chromatin accessibility
	•	down – decreased chromatin accessibility
	•	neutral – no predicted regulatory effect

```
python ./scripts/TabPFN_testSNP_classifier_pred.py --input_file CD4Naive_single_SNP_merged_features_classifier.csv
```
